Shadow DOM

  • Key benefit: encapsulation (protects JS and CSS references)

    • Nodes are not selectable via querySelector (by default)

Shadow DOM

Template use without shadow DOM

this.appendChild(templateClone);
this.counterDiv = this.querySelector("# deluxe - counter - content ");

Shadow DOM

Template use with shadow DOM

var root = this.createShadowRoot();
root.appendChild(templateClone);
this.counterDiv = root.querySelector("#deluxe-counter-content");

Content Projection

<div id="shadow-host">
    This is content.
</div>
<template id="template">
    <div id="shadow-div">
        <div>Inside the shadow root, baby!</div>
    </div>
</template>
<script>
    var shadowHost = document.querySelector("#shadow-host");
    var shadowRoot = shadowHost.createShadowRoot();
    var templateClone = document.importNode(
        document.querySelector("#template").content, true);
    shadowRoot.appendChild(templateClone);
</script>

Content Projection with Insertion Points

<div id="shadow-host">
    <h2>I am the host.</h2>
    <div class="content">
        And this is my content.
    </div>
</div>
<template id="template">
    <div id="shadow-div">
        <content select="h2"></content>
        <div>Inside the shadow root, baby!</div>
        <content select=".content"></content>
    </div>
</template>
<script>
    var shadowHost = document.querySelector("#shadow-host");
    var shadowRoot = shadowHost.createShadowRoot();
    var templateClone = document.importNode(
        document.querySelector("#template").content, true);
    shadowRoot.appendChild(templateClone);
</script>

Content Projection with Insertion Points

  • Content selectors can only select elements that are the root of the shadow host

  • If select matches more than one element, the first one wins

CSS Scoping

  • By default styles are scoped to the shadow root

  • CSS styles in the parent document don’t affect the shadow DOM for states like :hover and :active

  • ::host pseudo-element styles the host element

    • overrides other selectors, but not styles specified in the style attribute

  • ::shadow pseudo-element lets you style elements in the shadow root

  • Both psuedo-elements are supported by querySelector()

CSS Scoping

<style>
    .counter::shadow div {
        border: 2px solid black;
    }
</style>

Total CSS Control

To style across shadow DOM bounderies, use the /deep/ combinator:

x-tabs /deep/ x-panel {
  ...
}

body /deep/ .library-theme {
  ...
}

Can be used with querySelector() as well

Total CSS Control

For native elements that are implemented using Shadow DOM, this works too:

video /deep/ input[type="range"] {
  background: hotpink;
}
